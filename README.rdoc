== README

Некоторые пояснения:
Редактирование имени пользователя через Ajax сделал полностью руками, без применения
библиотеки best_in_place, основная причина в том, что на странице у одного пользователья может
быть несколько коммитов, соответствнно при редактировании его имени
в одном коммите, изменения должны произойти во всех коммитах этого пользователя.

Контроллеры для рессурсов, которые не являются моделями, в частности Import и Search
протестировал в основном по части редиректов и рендеров и сохранения данных в базу.
Помимо этого тотальное тестирование этих ресурсов сделано в Acceptance тестах.

Внес изименения.
 1. На Ajax серевер отдает json
 2. Переписал поиск, сделдал его как зависимую имплементацию экшена Index (если если в параметрах пришел email,
 то экшен отдает только его коммиты, если нет, то все). Тем самым оптимизировал запрос в бд для поиска.
 3.Добавил валидацию имени пользователя и репозитория. (не делал отдельного модуля для кастомных ошибок, поскольку
  приложение   тестовое, в реальных проектах конечно предусматриваю полную обработку возможных кастомных ошибок
  в отдельном моудуле)
 4. Оптимизировал запрос импорта в соответстви с лимитами API GitHub.  Перевернул полинтернета, единственным оптимальным
 способом для этого я счел постраниный импорт. (возожно есть более оптимальный вариант, буду рад, если кто-нибудь покажет)
 5. Протестировал импорт на всех уровнях (модель, контроллер, acceptance)

В качестве внешнего репозитория для тестирования ресурса Import я использовал один из своих
репозиториев. Поэтому в тестах можно увидеть Имя Rim-777 и  репозиторий "AskmeHow"
